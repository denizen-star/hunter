App Dash (Application Dashboard)

URL: /dashboard

When to Use: This is your primary dashboard for daily job application management. Use it daily to track application statuses, sort applications, view match scores, and navigate to application details.

Key Features

Tabbed Interface

The dashboard organizes applications into status tabs:

  • Active (default) - All applications except rejected
  • All - Complete list of all applications
  • Pending - Applications not yet submitted
  • Applied - Submitted applications
  • Contacted - Applications where you've contacted someone
  • Hiring Mgr - Applications where you contacted a hiring manager
  • Interviewed - Applications in interview process
  • Offered - Applications with job offers
  • Rejected - Rejected applications
  • Accepted - Accepted offers

Dynamic Counts: Each tab shows the number of applications in parentheses (e.g., "Applied (15)")

Advanced Sorting

Each tab has independent sorting options. Sort by:

  • Updated Timestamp: Newest/Oldest first - see most recently updated applications
  • Applied Timestamp: Newest/Oldest first - see most recently applied
  • Job Posted Date: Newest/Oldest first - prioritize fresher job postings
  • Match Percentage: Highest/Lowest first - focus on best matches or address weaker ones

Independent Sorting: Each tab remembers its own sort preference. Sort "Applied" by Match % while keeping "Interviewed" sorted by Updated date.

Application Cards

Each application is displayed as a card showing:

  • Company Name & Job Title
  • Match Score: Prominently displayed percentage (e.g., "85%")
  • Status Badge: Color-coded pill showing current status
  • Metadata: Applied Date, Posted Date (ðŸ“‹ icon), Last Updated Date
  • View Summary Button: Click to open detailed application summary page

Statistics Cards

Top of dashboard shows quick stats:

  • Total Applications
  • Pending Count
  • Applied Count
  • Interviewed Count
  • And more status-specific counts

Why Use App Dash?

  • Daily Management: Your go-to page for day-to-day application tracking
  • Quick Navigation: Fast access to any application via organized tabs
  • Status Focus: Quickly see what needs attention in each status category
  • Match Score Visibility: Instantly see which applications have best match scores
  • Flexible Sorting: Organize applications how you need them for different tasks

How to Use App Dash

  1. Start with Active Tab: This shows all in-flight applications (everything except rejected) - your main working view
  2. Review Status Tabs: Check each status tab to see applications in different stages
  3. Use Sorting: Sort by Match % to prioritize high-scoring applications, or by Updated to see what needs attention
  4. Click Cards: Click "View Summary" on any card to see full application details
  5. Track Progress: Move applications through tabs as their status changes

Network Dash (Networking Dashboard)

URL: /networking

When to Use: Use daily or weekly to manage networking contacts, track relationship progress, identify contacts needing follow-up, and review networking pipeline.

Key Features

Status Filters

Filter contacts by their current relationship status:

  • To Research: Contacts identified but research not complete
  • Contacted: Initial outreach sent (LinkedIn connection or email)
  • In Conversation: Actively exchanging messages
  • Meetings: Meetings scheduled or completed
  • New Connection: Successful initial conversation complete
  • Nurture: Ongoing relationship maintenance
  • Referral Partner: Strong advocates and referral sources
  • Cold/Archive: Inactive contacts

Timing Filters

Filter by days since last update to identify contacts needing attention:

  • All: Show all contacts regardless of timing
  • Recent: Updated in last 7 days (green border)
  • Needs Activity: 8-15 days since update (yellow border)
  • Approaching: 15-30 days since update (yellow border)
  • Overdue: 30-60 days since update (red border) - for "Contacted" status
  • Nurture: 60-90 days (blue border) - scheduled future contact
  • Cold: 90+ days (gray border) - for prospecting statuses

Contact Tiles

Each contact displayed as a tile showing:

  • Person Name & Company: Contact identification
  • Job Title: Their current position
  • Status Badge: Fintech A co relationship status
  • Match Score: Skill overlap percentage with your resume
  • Timing Indicator: Color-coded left border based on days since update
  • Next Step: Suggested action based on status and timing
  • Dates: Created date and last updated timestamp
  • View Button: Click to open full contact summary page

Create New Contact

"New Contact" button to quickly add networking contacts from the dashboard.

Why Use Network Dash?

  • Relationship Management: Track all professional relationships in one place
  • Follow-up Tracking: Timing filters help identify contacts needing attention
  • Status Overview: Quickly see where each relationship stands
  • Pipeline Management: Manage networking pipeline similar to job application pipeline
  • Color-Coded Urgency: Visual indicators show which contacts need immediate attention

How to Use Network Dash

  1. Choose Status Filter: Select the relationship phase you want to focus on
  2. Apply Timing Filter: Use timing filters to find contacts needing follow-up
  3. Review Contact Tiles: Check match scores and next step suggestions
  4. Take Action: Click contacts with overdue timing or high match scores
  5. Update Status: Move contacts through status filters as relationships progress
  6. Create New Contacts: Use "New Contact" button to add contacts from company research

Progress Dash (Progress Dashboard)

URL: /progress

When to Use: Use weekly or bi-weekly for strategic pipeline reviews. Perfect for assessing overall progress, identifying bottlenecks, understanding conversion rates between stages, and planning next steps.

Key Features

Unified Progress View

The Progress Dashboard provides a comprehensive view of your entire job search pipeline, showing:

  • Pipeline Stages: Visual representation of applications moving through stages
  • Conversion Metrics: How many applications convert from each stage to the next
  • Bottleneck Identification: See where applications tend to get stuck
  • Progress Trends: How your pipeline health changes over time

Stage Analysis

Analyze progress through key stages:

  • Application Submission: How many applications you've submitted
  • Response Rate: Percentage receiving company responses
  • Interview Conversion: How many responses lead to interviews
  • Offer Rate: Interview-to-offer conversion

Why Use Progress Dash?

  • Strategic Planning: Understand where to focus efforts based on conversion rates
  • Bottleneck Detection: Identify stages where applications stall
  • Progress Tracking: See overall health and momentum of your job search
  • Goal Setting: Set realistic targets based on conversion metrics
  • Weekly Reviews: Perfect for weekly pipeline health checks

How to Use Progress Dash

  1. Review Pipeline Health: Check overall progress and conversion rates
  2. Identify Bottlenecks: Look for stages with low conversion rates
  3. Assess Trends: See if your pipeline is improving over time
  4. Plan Actions: Use insights to focus efforts on areas needing improvement
  5. Set Targets: Establish goals for each pipeline stage

Archive Dash (Archived Dashboard)

URL: /archived

When to Use: Use occasionally to review rejected applications, analyze rejection patterns, or reference past applications. Helps keep main dashboard focused on active opportunities.

Key Features

Rejected Applications

The Archive Dashboard shows all applications with "Rejected" status, including:

  • Complete Application History: All rejected applications for reference
  • Same Card View: Same application cards as main dashboard for consistency
  • Full Details Access: Can still view full summaries and documents
  • Search & Sort: Sort and organize archived applications like active ones

Why Use Archive Dash?

  • Keep Main Dashboard Clean: Separates active pipeline from closed opportunities
  • Pattern Analysis: Review rejections to identify patterns or common issues
  • Historical Reference: Access past applications and documents when needed
  • Focus on Active: Main dashboard stays focused on opportunities still in play
  • Learning Opportunities: Review rejections to improve future applications

How to Use Archive Dash

  1. Review Rejections: Periodically check archived applications
  2. Analyze Patterns: Look for common themes in rejected applications
  3. Reference Past Work: Use archived applications as reference for similar roles
  4. Learn & Improve: Identify what might have led to rejections to improve future applications

Note: Applications are automatically moved to Archive Dash when their status is set to "Rejected". You don't need to manually archive them.

Dashboard Workflow

Daily: Use App Dash

Start each day with App Dash to see all active applications. Use the "Active" tab by default to focus on in-flight opportunities. Sort by Updated date to see what needs attention.

Daily: Check Network Dash

Review Network Dash using timing filters to identify contacts needing follow-up. Focus on "Overdue" or "Needs Activity" contacts to maintain relationships.

Weekly: Review Progress Dash

Weekly review of Progress Dash to assess pipeline health, identify bottlenecks, and plan strategic improvements for the coming week.

Occasionally: Check Archive Dash

Periodically review Archive Dash to analyze rejection patterns and learn from past applications.

Pro Tip: Set App Dash as your browser homepage or bookmark it. Use Network Dash daily for relationship management. Reserve Progress Dash for strategic weekly reviews. Keep Archive Dash for occasional analysis.
